Création d’un utilisateur

Consigne :

Créer un utilisateur nommé “testuser”, dans un groupe “testgroup” avec la commande adduser. Cet utilisateur ne doit pas avoir les droits d’administrateur.

Explication télégraphique :

1/ Créer le groupe testgroup

$

sudo groupadd testgroup

2/ Créer l’utilisateur testuser et l’attribuer au groupe testgroup

$

sudo adduser testuser --ingroup testgroup

3/ Saisir un mot de passe pour l’utilisateur.

4/ Le retaper.

5/ Entrer les informations demandées ou appuyer sur entrer pour laisser les valeurs par défaut.

6/ Confirmer les informations en tapant “O”, ou éditer-les si besoin en tapant “n”.

Détails & Explications :

Tout d’abord, comme nous souhaitons associer l’utilisateur à un groupe qui n’est pas encore créé, pour des raisons de facilité, il vaut mieux d’abord créer le groupe. Cette solution nous permet simplement de définir le groupe lors de la création de l’utilisateur sans avoir à créer l’utilisateur, puis le groupe pour ensuite les associer. Cette méthode est possible car la commande de création d’un utilisateur nous permet directement de faire l’association lui et le futur groupe associé. Afin d'exécuter la commande avec les permissions, il ne faut pas oublier “sudo” avant celle-ci.

$

sudo groupadd testgroup

Une fois le groupe créé, il faut également créer l’utilisateur en l’associant au groupe suivant : “testgroup”. Afin de réaliser la création de l’utilisateur, nous avons fait le choix d’utiliser la commande “adduser”. Comme pour la commande précédente, il faut utiliser le “sudo” pour être sûr d’avoir la permission. l’argument “--ingroup GROUP” (avec GROUP, le nom d’un groupe) qui peut être remplacé par “--gid ID”(avec ID, l’identifiant d’un groupe) permet de définir le groupe auquel l’utilisateur va être associé.

 

$

sudo adduser testuser --ingroup testgroup

A présent, il ne reste plus qu’à entrer le mot de passe pour l’utilisateur “testuser” et à le confirmer une seconde fois. Pour des mesures de sécurité, celui-ci doit contenir plus de 8 caractères, de plus le système demande un minimum de non-redondance de celui-ci. Le système demande ensuite d’entrer des informations sur l’utilisateur tel que son nom, son prénom, son numéro de téléphone. Il est possible de mettre une valeur par défaut en appuyant sur ENTRER. Enfin, confirmer la création de l’utilisateur avec “O” sinon faite “n” pour annuler. Maintenant, le groupe “testgroup” est bien créé et s’est vu associer l’utilisateur “testuser”.

Installation des logiciels :

Consigne :

Installer les logiciels suivants :

  • emacs
  • apache2
  • wireshark
  • javac et java

Explication télégraphique :

1/ Mettre à jour les dépôts et les paquets

$

sudo apt update

$

sudo apt upgrade -y

2/ Installer emacs et Apache2

$

sudo apt install emacs -y

$

sudo apt install apache2 -y

3/ Installer wireshark

$

sudo apt install wireshark -y

4/ À l’apparition du panel de configuration de wireshark lors de son installation, sélectionner “Oui” avec les flèches directionnelles et appuyer sur “entrer”.

5/ Ajouter “testuser” au groupe “wireshark” pour l’autoriser à capturer des paquets

$

sudo usermod -aG wireshark testuser

6/ Installer javac et java

$

sudo apt install openjdk-17-jdk -y

Détails & Explications :

Tout d’abord, il faut réaliser les mises à jour des dépôts et des paquets de la machine. Pour celà, on utilise la commande suivante :

$

sudo apt update

$

sudo apt upgrade -y

La première commande “apt update” est généralement assez rapide contrairement à la seconde qui peut prendre une dizaine de minutes. L’argument “-y” permet de mettre automatiquement “O” à la question qu’il va poser pour valider la mise à jour. Elle n’est pas obligatoire mais fortement conseillée pour gagner un peu de temps.

Une fois les paquets et dépôts mis à jour, il va falloir commencer à installer les différents programmes. On va commencer par “emacs” avec la commande :

$

sudo apt install emacs -y

Une fois le téléchargement terminé de emacs, on peut considérer que celui-ci est complètement installé.

On va ensuite installer “apache2” qui est un client web permettant d’héberger un site sur sa machine linux au même titre que “nginx”. Afin de l’installer, on peut utiliser la commande suivante :

$

sudo apt install apache2 -y

Comme “emacs”, “apache2” n’a pas besoin de configuration supplémentaire pour fonctionner. Maintenant que Apache2 est installé, on peut passer à WireShark.

Pour WireShark, il y a quelques manipulations supplémentaires afin de répondre au besoin que tous les utilisateurs doivent pouvoir réaliser une capture réseaux avec WireShark.

$

sudo apt install wireshark -y

Lors de l’installation, une fenêtre va apparaître sur votre écran, il ne faut surtout pas faire entrer. Vous devez d’abord sélectionner “oui” à l’aide des flèches directionnelles puis faire ENTRER. Cette manipulation permet à l’ensemble des utilisateurs de réaliser une capture réseau. Une fois l'installation terminée, il faut encore ajouter l’utilisateur dans un groupe wireshark qui l’autorise à réaliser des captures réseaux, avec la commande suivante :

$

sudo usermod -aG wireshark testuser

Enfin, on va procéder à l’installation de “java & javac” avec le package qui se nomme “openjdk-17-jdk”. Cette installation est aussi simple que celle de “emacs” ou “apache2”. Il suffit d'exécuter la commande suivante :

$

sudo apt install openjdk-17-jdk -y

L’utilisation de l’argument “-y” permet toujours de confirmer automatiquement que l’on souhaite installer “openjdk”. Maintenant, nous avons terminé est installation et nous nous trouvons avec un système similaire à celui-ci dessous.

Installer les scripts de gestion d’une poubelle fait en cours (srm, lkt, clt, rft) dans un répertoire du système (par exemple /scripts)

Explication télégraphique :

1/ Exécuter la commande suivante

$

sudo mkdir /scripts && sudo wget -O /scripts/scripts.zip https://github.com/zonetecde/SAE-S1-03/releases/download/v1.0.0/scripts.zip && sudo unzip /scripts/scripts.zip -d /scripts && sudo rm /scripts/scripts.zip && sudo chmod -R a+x /scripts/*

Détails & Explications :

Afin de faciliter l’installation de nouvelles machines sans avoir à réaliser une multitude de commandes, nous avons opté pour une installation des programmes depuis un dépôt sur GitHub. L’ensemble de nos fichiers contenant nos commandes ont donc été publiés sur notre GitHub puis grâce à un wget, nous pouvons directement les installer sur la machine.

Cette commande commence par créer le répertoire “scripts” dans la racine de notre machine.

$

sudo mkdir /scripts

Elle télécharge ensuite l’ensemble des scripts qui sont contenus dans un fichier ZIP. Le fichier ZIP provient du repo GitHub que nous avons créé. Celui-ci est directement mis dans le dossier “scripts”.

$

sudo wget -O /scripts/scripts.zip https://github.com/zonetecde/SAE-S1-03/releases/download/v1.0.0/scripts.zip

Il faut ensuite décompresser l’archive ZIP grâce à la commande

$

sudo unzip /scripts/scripts.zip

Enfin, on peut supprimer le fichier ZIP qui ne sert plus à rien et il faut également penser à mettre les permissions pour que les scripts puissent être exécutés par les utilisateurs. Ces deux actions sont réalisé par les commandes suivantes :

$

sudo rm /scripts/scripts.zip

$

sudo chmod -R a+x /scripts/*

Faire en sorte que le serveur Web apache2 se lance automatiquement au démarrage de la machine en utilisant la gestion des services avec systemd. Et faire en sorte que la page web accessible avec l’url : http://127.0.0.1 affiche une version HTML de votre notice technique

Détails & Explications :

Par défaut, le serveur Apache se lance automatiquement au démarrage de la machine après son installation. Cependant, si ce n’est pas le cas, on doit manuellement activer le service relié à cet événement à l’aide de la commande :

systemctl enable apache2

cd /var/www

$

sudo rm html -r

Configurer une rotation du fichier log

Consigne :

Configurer une rotation du fichier de log /var/log/install.log en utilisant logrotate, de sorte que le fichier soit compressé dès qu’il dépasse 200 octets, et qu’il y en ait 5 sauvegardes maximum.

Explication télégraphique :

1/  Créer un fichier de configuration

$

sudo mkdir ~/logrotate_config

2/ Ajouter la configuration suivante dans ce fichier

$

sudo nano ~/logrotate_config/config

3/ Limite la taille à 200 octets avant compression et 5 sauvegardes maximum

1

su root root

/var/log/install.log {

     size 200k

     compress

     rotate 5

}

4/ Rotation du fichier “config”

$

sudo logrotate -f ~/logrotate_config/config

Script & Explications :

Pour faire en sorte que le fichier /var/log/install.log soit automatiquement compressé dès qu’il dépasse 200 octets, et qu’il y en ait 5 sauvegardes maximum, nous devons créer un fichier de configuration utilisé par la suite par logrotate.

Nous avons décidé de créer ce fichier dans le dossier ~/logrotate_config et l’avons appelé config.

La commande suivante permet de créer le dossier “logrotate_config” dans la racine de l’utilisateur.

$

sudo mkdir ~/logrotate_config

Ensuite, on utilise la commande “nano” qui nous permet de “touch” et d’ouvrir le fichier (comme emacs). On peut ensuite y coller la configuration qui est présente quelques lignes plus haut.

$

sudo nano ~/logrotate_config/config

Il ne nous reste plus qu’à appliquer la rotation grâce à la commande suivante :

$

sudo logrotate -f ~/logrotate_config/config
